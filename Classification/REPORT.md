# Практическая работа 2. Классификация данных

## Цель работы

Научиться выполнять классификацию данных на основе вектора признаков.

## Описание датасета

В работе использован датасет `public_data.csv` с признаками веб-страниц для определения класса сайта.

- Количество объектов: 9949.
- Количество столбцов: 32.
- Идентификатор строки: `Row_id`.
- Целевой столбец: `Result`.
- Классы: `-1` и `1`.
- Количество признаков для обучения: 30.

Столбец `Row_id` не использовался как признак, так как это технический идентификатор строки.

## Выбранный метод классификации

Для итоговой модели выбран `RandomForestClassifier`.

Случайный лес строит несколько решающих деревьев и объединяет их ответы голосованием. Метод хорошо подходит для табличных данных, устойчив к шуму и не требует масштабирования признаков, что удобно для датасета с дискретными значениями `-1`, `0` и `1`.

Для сравнения были проверены несколько моделей:

| Метод | Accuracy |
| --- | ---: |
| DecisionTreeClassifier | 0.96785 |
| RandomForestClassifier | 0.97588 |
| ExtraTreesClassifier | 0.97588 |
| GradientBoostingClassifier | 0.94855 |
| HistGradientBoostingClassifier | 0.96825 |
| KNeighborsClassifier | 0.94333 |
| SVC | 0.96664 |

## Возникшие трудности и решения

Главная особенность задания состоит в том, что проверочный файл не содержит целевой столбец `Result`. Поэтому программа обучает модель на размеченном `public_data.csv`, проверяет наличие всех нужных признаков во входном файле и добавляет предсказанный столбец `Result` в выходной CSV.

Также из обучения был исключен `Row_id`, потому что использование идентификатора может привести к переобучению и не отражает реальные свойства объекта.

## Ответы на контрольные вопросы

1. Классификация применяется для задач, где нужно отнести объект к одному из заранее известных классов: фильтрация спама, диагностика, распознавание изображений, определение фишинговых сайтов, скоринг клиентов.

2. Обучающая выборка нужна для настройки модели, а тестовая выборка нужна для независимой проверки качества на данных, которые модель не видела при обучении.

3. Для классификации применяются решающие деревья, случайный лес, метод ближайших соседей, логистическая регрессия, метод опорных векторов, градиентный бустинг, наивный байесовский классификатор и нейронные сети.

4. При классификации могут использоваться очистка данных, обработка пропусков, кодирование категориальных признаков, масштабирование числовых признаков, удаление неинформативных столбцов, балансировка классов и разделение данных на обучающую и тестовую выборки.

5. Для оценки качества классификации применяются `accuracy`, `precision`, `recall`, `F1-score`, матрица ошибок, `ROC-AUC` и `PR-AUC`.
